Requirements

See also README

For a programming project the computer is set up in the same way as if any text editor is used for coding and the tested program would be run from a command line. Presently, programming projects can be defined for Java and Perl.

To compile Java files by simply clicking the 'compile' button or selecting the corresponding option in the menu it must be made sure that the program is run using the Java Runtime environement (JRE) contained in the Java Development Kit (JDK).

Running a project

Types of projects

Projects of the following types can be assigned to the program:

  • Java (Testing Java code and creating a jar file)
  • Perl (Testing a Perl script)
  • HTML (Viewing HTML code in a browser)

Assign files as project

  1. Open a file from or save a new file to the directory that is the supposed project root directory. The file may also be located in a sub-directory of that root. Note that in the simple case that the project does not incolve any particular directory structure the project root is just the location of the project file.
  2. Open the settings window by selecting 'Project' > 'Assign as project'.
  3. Enter the name of the main source file (this applies to Java and Perl but not Html). The main file does not have to be the opened file but must exist.
  4. If the main file is part of a package (applies to Java) enter the package name (or the package path) in the field 'Package containing the main class'.
  5. If source files (or packages) are saved in a separate sub-directory of the supposed project root the name of this sources directory is entered in the field 'Name of sources directory'.
  6. The name of a separate destination directory for compiled files (applies to Java) maybe entered in the field 'Name of executables directory'. This folder will be created in the project's root directory when source files are compiled.
  7. Click ok. If entries are correct, that they conform to a valid file path, the project files can be viewed in the 'Project explorer'. This is opened by selecting 'View' > 'Project explorer'.
  8. The field 'Name of project root' does not have to be filled unless the project category is Html. For Html only the project location but no project file has to be specified.

Changing between projects

  • 'Project' > 'Assign as project' is selected to assign an additional project. The file that is supposed to be part of the new project must be in the selected tab.
  • Changing between already set projects is done by selecting a file that is part of the project to be set active and then selecting 'Project' > 'Change project' or, to additionally view or modify the settings, 'Project' > 'Project settings'.

Retrieving projects

  • The project that was active when the program was closed the last time is retrieved and set active when a file of that project is opened (or a new file is saved to that project) and no other project is already active.
  • If 'Save configuration in the project root' is selected in the settings window the parameters for the project are stored in a properties file that is saved in the project's root folder. It can be deleted at any time by unselecting the option to save it.
    This properties file in a way marks a project as an "Eadgyth project". If a file of a marked project is opened (or a new file is saved to that project) the project is set active even if it was not active lastly. Also, if another project is already active, a marked project is loaded and can be set active by selecting 'Project' > 'Change project'.

Setting the language

  • The language is defined based on the extension of an opened or newly saved file.
  • Selecting the language in the Edit menu ('Edit' > 'Language') sets the default for new empty files.

Testing Java code and creating a jar file

  1. Compile the source file(s) by selecting either 'Save selected source file and compile' or 'Save all source files and compile' in the 'Project' menu. If errors occured it is asked to open the console panel to view messages. Note that only files in the assigned project are included if this feature of the program is used.
  2. If the compilation was successful the Java program can be run for testing by selecting 'Run' in the 'Project' menu. The output and error messages from the tested program are displayed in the console panel. If the tested program asks for input through a command line this is entered in the text area of the console panel. A tested program can be forcibly quit by clicking the 'stop' symbol in the tool bar of the console panel.
  3. Arguments for a start script may be entered in the text field 'Arguments' (separated by single spaces) in the settings window ('Project' > ' Project settings').
  4. An executable java archive (jar) may be created by selecting 'Project' > 'Create jar'. The jar file is saved in the executable's folder or, if no such folder is defined, in the project's root folder. A name for the jar file can be specified in the settings window. If no name is specified the name of the main class is used. Note that presently only files found in the assigned project are bundled in a jar when the 'Create jar' option of the program is selected.
  5. To include files other than java (or class) files in a compilation and in a jar file the extensions of these files would be specified in the text field 'Included file types'. The files are searched in the sources directory (in a compilation) and in the executables directory (for creating a jar) if these directories are defined in the assigned project.

Testing a Perl script

  1. Save changes to the script and select 'Run' in the 'Project' menu.
  2. Any output (error) is shown in the console panel. NOTE: there is a limitation when the script asks for input through a command line. In that case any output from the script expected before the user input is asked for (like "Enter your name:") is not printed (more precisely it's printed only after all inputs were made). The output would be shown as expected (as far as tested) if the autoflushing of Perl's STDOUT is turned on in the script itself. The other possibilty is, of course, to start the command line/terminal of the OS to test the script (see Using the console).
  3. Arguments for a start script may be entered in the text field 'Arguments' (separated by single spaces) in the settings window ('Project' > 'Project settings').

Viewing HTML code in a web browser

  1. Save changes to the selected html file and select 'Run' in the 'Project' menu to view the html file in the default web browser.

Using the console

The console area shall show messages after compilation of a project and the output of a program that is tested.

In addition, self-chosen system commands entered in a dialog can be run in the current working directory (the project's root directory).

A run process can be forcibly quit by clicking the 'stop' symbol.

The console does not emulate the terminal of the operating system! However, it is possible (or rather not prohibited) to run the system's terminal in the console by running the command that starts the terminal (e.g., cmd.exe in Windows). It is important to note that a tested program thereupon started from this terminal CANNOT be forcibly quit by clicking the 'stop' symbol.

Hint: using the command to start "THE" terminal (in Windows this would be cmd.exe /c start) presets the working directory for the terminal to the project's root directory.

To run a system command...

  1. Open the console panel in the menu 'View' > 'Console'.
  2. Press the button 'Cmd...' and enter a command in the shown dialog window. Klick ok in the dialog window to run the command.
  3. To run a previous command press the '>' button; to change a command open the command dialog again by clicking 'Cmd...'.

Using the project explorer

The project explorer is opened by selecting 'View' > 'Project explorer'. It shows the file system of a project once one is assigned.
The explorer always shows the project that is currently set active. A few operations are supported:

  • The project's file system is shown as a tree in which folders are expanded or shrunk by clicking the (+), (-) handles, as usual. However, a double left click on a folder symbol does not expand/shrink the folder but displays a new tree rooted at the clicked folder. To move back the 'Folder up' symbol is clicked. The topmost level is always the project's root directory.
  • A double left click on a file opens the clicked file in a new tab.
  • A file may be deleted after a right click on a file symbol and selecting'delete'. If the deleted file is in an open tab, the tab is not closed but goes into "Save as" mode.
  • A folder may be as well deleted after a right click on a folder symbol but a folder is protected from deletion if it is not empty. The exception is, if present, the folder for executable files or its subfolders. This may be deleted for a completely renewed compilation.
  • A new folder may be created after a right click on a folder symbol and selecting 'Create new folder'.
  • Changes to the project's file structure made from within the program should be updated in the file view. Changes made from outside the program would be updated "manually" by clicking the 'refresh' symbol.
