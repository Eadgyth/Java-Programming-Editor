Requirements

The Java runtime environment must be version JRE 8 to run Eadgyth.

For a Java project it is required that a Java development kit (JDK) is installed and the computer is set up to use the JDK. For the JDK of Oracle, see: Oracle JDK. Also, the filepath of the JDK must be saved in the ‘settings’ file. This, if not done already, the program asks for the first time it is tried to compile source files.

Configuring and testing a Java project

Project settings

The setting of a project always starts from an existing file that is opened (or newly saved) and is in the selected tab. The file is found in a directory that is supposed to be the project's root or in sub-directories of this root, i.e. a separate directory for sources and/or a package.

  1. Open a java source file or save an unnamed tab as a java source file.
  2. Open the settings window using ‘Project settings’ in the menu ‘Project’.
  3. Enter the name of the source file that contains the main method in the text field ‘Name of main program file’ (without extension). The main file does not have to be the opened file.
  4. If the main file is part of a package enter the package name in the text field ‘Package with main class’. If it is located in a sub-package enter the path with normal file separators (slash in Windows).
  5. If the directory that contains source files (or packages) is not supposed to be the project's root but a separate sources directory the name of this directory is entered in text field ‘Sources directory’.
  6. A separate destination directory for compiled class files maybe entered in the text field ‘Executables directory’ (the folder will be created during compilation).
  7. Click ok. If entries are correct the project can be viewed in the ‘Project explorer’ that is opened in the ‘View’ menu.
Compiling and running a project
  1. Compile the source file(s) using ‘Save all and compile’ in the ‘Project’ menu. If errors occur you will be asked to open the console to view messages.
  2. If the compilation was successful the new program can be run for testing by using ‘Run’ in the ‘Project’ menu (this will also open the console panel). The tested program can be forcibly quit using the ‘stop’ button of the console panel.
  3. Arguments for a start script may be entered in the text field ‘Arguments’ (separated by single spaces) in the Settings window
  4. A jar file may be created by using ‘Build’ in the ‘Project’ menu.
Changing between projects
  • A new project can be set by opening a file that is not part of the current project and then opening the settings window (‘Project’ > ‘Project settings’). The new file must be in the selected tab.
  • Changing between already set projects is done by selecting a file that is part of the project to be set active and using ‘Project’ > ‘Change project’.
  • A project maybe also defined for a txt file (which only allows viewing files in the project explorer) or for a HTML file which can be shown in Web browser by using ‘Run’ in the ‘Project’ menu. The keyword coloring for html code is rudimentary though.

Using the console

The console area shall show messages after compilation of a project and the output of a program that is tested. Interactive programs can be run too, at least Java programs.

In addition, self-chosen system commands entered in a dialog can be run in the current working directory. For example, a created jar file may be tested by the command java -jar SomeProgram.jar. A run process can be forcibly quit using the ‘stop’ button.

Note that the console is not a terminal so that terminal commands are initially invalid. While it is possible to run the terminal in the console (by using the system command that starts the terminal) a program thereupon started from this terminal cannot be forcibly quit using the ‘stop’ button.

To run a system command...
  1. Open the console panel in the menu ‘View’.
  2. Press the button ‘Cmd...’ and enter a command in the shown dialog window. Click ok in the dialog window to run the command.
  3. To run a previous command press the ‘>’ button; to change a command open the command dialog again by pressing the ‘Cmd...’ button.
Further options
  • A new instance of Eadgyth can be run from the console by pressing the button with Eadgyth icon. One reason to do this would be to view the output from a tested new plugin.
