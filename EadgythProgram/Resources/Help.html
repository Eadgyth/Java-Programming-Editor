Eadgyth help (still in progress)

Requirements

The Java runtime environment must be version JRE 8 to run Eadgyth.

Configuring a java project

It is required that a Java development kit (JDK) is installed and the computer is set up to use the JDK. For the JDK of Oracle, see:
Oracle JDK

  1. Open a java source file or save an unnamed tab as a java source file. Only at the first time you will be asked to enter the filepath of the JDK.
  2. Open the settings window using ‘Project settings’ in the menu ‘Project’.
  3. Enter the name of the source file that contains the main method in the text field ‘Name of main program file’ (without extension).
  4. If this file is within a package enter the package name in the text field ‘Package with main class’. If it is located in a sub-package enter the path with normal file separators (slash in Windows).
  5. A separate directory for source files (or packages) may be entered in the text field ‘Sources directory’.
  6. A separate destination directory for compiled class files maybe entered in the text field ‘Executables directory’ (the folder will be created during compilation).
  7. Click ok. If entries are correct the project can be viewed in the ‘Project explorer’ that is opened in the ‘View’ menu
  8. Compile the source file(s) using ‘Save all and compile’ in the ‘Project’ menu. If errors occur you will be asked to open the console to view messages.
  9. Run the project using ‘Run’ in the ‘Project’ menu (this will also open the console panel).
  10. A jar file may be created by using ‘Build’ in the ‘Project’ menu.
Further options
  • Arguments for a start script may be entered in the text field ‘Arguments’ (separated by single spaces).
  • The name of a build file (jar file in the case of java) can be specified in the field ‘Name for jar file’. If the field is not filled, the jar name is the name of the java file that contains the main method.
  • The project can be changed by opening a file that does not belong to the current project.
  • An existing project configuration will be retrieved upon opening any file of a project previously worked on.
  • A project maybe also defined for a txt file (which only allows viewing files in the project explorer) or for a HTML file which can be shown in Web browser using ‘Run’ in the ‘Project’ menu. The keyword coloring for html code is rudimentary though.

Using the console

The console area shall show messages after compilation of a project and the output of a program that was started as in step 9 (see above). Interactive programs can be run too, but it is not clear yet if this works in all situations.

In addition, self-chosen system commands entered in a dialogue maybe run in the current working directory (the root directory of a project). For example, a created jar file may be tested by the command java -jar SomeProgram.jar. A run process can be forcibly quit using the ‘stop’ button.

Note that the term ‘system command’ does not mean that it is possible to run directly terminal commands. While it is possible to run the terminal in the console (by using the system command that starts the terminal) a program thereupon started from this terminal cannot be forcibly quit using the ‘stop’ button.

To run a system command...
  1. Open the console panel in the menu ‘View’.
  2. Press the button ‘Cmd...’ and enter a command in the shown dialog window. Click ok in the dialog window to run the command.
  3. To run a previous command press the ‘>’ button; to change a command open the command dialog again by pressing the ‘Cmd...’ button.
Further options
  • A new instance of Eadgyth can be run from the console by pressing the button with Eadgyth icon. One reason to do this would be to view the output from a tested new plugin.
